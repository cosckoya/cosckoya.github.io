# Deployment and CI/CD Guidelines

## GitHub Actions Pipeline

**Workflow**: `.github/workflows/gh-pages.yml`

**Trigger**: Push to `main` branch only

**Pipeline Steps**:
1. Checkout code (actions/checkout@v4)
2. Configure Git credentials (Roy Batty user)
3. Setup Python 3.x
4. Cache MkDocs Material assets (weekly rotation)
5. Cache pip dependencies (based on requirements.txt hash)
6. Install dependencies: `pip install -r requirements.txt`
7. Build site: `mkdocs build --strict`
8. Deploy: `mkdocs gh-deploy --force` to `gh-pages` branch

**IMPORTANT**: Only pushes to `main` trigger deployment

## Branch Strategy

- `main` - Production branch (auto-deploys to GitHub Pages)
- `develop` - Development base branch
- Feature branches - Create from develop, PR back to develop
- Only maintainer pushes to main

## Local Deployment Testing

```bash
make build       # Build in strict mode locally
make serve       # Test with live reload at localhost:8000
make deploy      # Manual deploy to GitHub Pages (rarely needed)
```

**Never** manually deploy unless CI/CD is broken.

## Build Artifacts

Generated artifacts (gitignored):
- `site/` - Built HTML/CSS/JS output (11MB)
- `.cache/` - MkDocs Material cache
- `__pycache__/` - Python bytecode
- `requirements-lock.txt` - Generated by `make update`

## Dependency Updates

```bash
make update      # Update all Python packages
make test        # Verify everything works
```

Dependabot runs monthly for automated dependency PRs.

## Deployment Checklist

Before merging to main:
- [ ] `make test` passes (pre-commit + strict build)
- [ ] `./scripts/docmaster-tools.sh full-maintenance` passes
- [ ] All navigation links verified with `make serve`
- [ ] PR reviewed and approved
- [ ] CI/CD green on feature branch
