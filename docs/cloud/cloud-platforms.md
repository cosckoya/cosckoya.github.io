# Cloud Platforms

Infrastructure and services provided by major cloud providers (AWS, Azure, GCP) enabling on-demand computing resources, storage, networking, and managed services. Organizations leverage these platforms to build, deploy, and scale applications without maintaining physical infrastructure.

______________________________________________________________________

## Core Concepts

=== "Infrastructure as a Service (IaaS)"

    **Definition:** Virtualized computing resources delivered over the internet, providing fundamental building blocks like virtual machines, storage, and networking.

    **Key Points:**

    - Complete control over infrastructure configuration
    - Pay-per-use pricing model
    - Examples: EC2 (AWS), Virtual Machines (Azure), Compute Engine (GCP)

=== "Platform as a Service (PaaS)"

    **Definition:** Development and deployment environment in the cloud with tools to build, test, and deploy applications without managing underlying infrastructure.

    **Key Points:**

    - Focus on application development, not infrastructure management
    - Built-in scaling and load balancing
    - Examples: Elastic Beanstalk (AWS), App Service (Azure), App Engine (GCP)

=== "Serverless Computing"

    **Definition:** Execution model where cloud provider dynamically manages allocation and provisioning of servers, charging only for actual compute time used.

    **Key Points:**

    - Zero server management
    - Automatic scaling
    - Event-driven architecture
    - Examples: Lambda (AWS), Functions (Azure), Cloud Functions (GCP)

=== "Multi-Cloud Strategy"

    **Definition:** Using multiple cloud providers to avoid vendor lock-in, optimize costs, and leverage best-of-breed services from each platform.

    **Key Points:**

    - Risk distribution across providers
    - Flexibility in service selection
    - Increased complexity in management
